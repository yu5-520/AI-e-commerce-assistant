# V13.1 Agent-Guided Snapshot Task Handoff

## 目标

把经营快照站和任务判断线连接起来，但不恢复旧的导入后直接生成任务 Hook。

## 核心原则

系统给事实，RAG给参照，Agent做判断。V13.1 只建立经营快照到任务判断线的轻量交接，不由系统规则直接生成任务。

## Handoff 状态

V13.1 默认生成：

```text
status = pending_agent_judgment
decision_status = awaiting_rag_agent
created_task_count = 0
```

意思是：经营快照已经可以进入任务判断线，但任务是否生成，需要后续 RAG Context 与 Agent Judgment 站点决定。

## 前端更新

导入报表后，前端刷新经营快照和 pipeline 状态，并调用 snapshot-task handoff，显示本次报表已经进入任务判断线。

## 不做的事

- 不恢复旧 Hook。
- 不让系统规则直接生成任务。
- 不把页面刷新变成任务生成入口。
- 不绕过 RAG 和 Agent 判断。

## 后续版本


由 RAG 和 Agent 判断是否真的生成任务快照，再进入任务池和任务生命周期。
