# V12.13.1 Legacy Trigger Cleanup

## 目标

删除 V12.13 之后仍然占流程位置的旧触发点，让主流程真正变成分站接力。

## 删除/禁用的旧影响

### 1. 导入接口不再同步生成任务

`src/api/routes/data_import.py`

- 删除上传链路里的任务生成副作用。
- 不再在上传请求里调用 `generate_risk_tasks_for_signals`。
- `_attach_v62_trend_and_risk_sync` 保留为兼容空函数，但不会生成任务。
- 导入接口只写：
  - import_uploaded
  - report_parsed
  - metric_facts_ready
  - operating_objects_ready
  - operating_unit_snapshot_ready

### 2. 旧导入闭环不再挂主链路

禁用上传请求内的旧 hook：

- `attach_v104_import_sync`
- `attach_v107_operating_profile`
- `attach_v108_tag_change_tasks`
- `attach_v116_import_closed_loop`

这些不再在导入接口里生成旧任务或旧标签闭环。

### 3. 前端导入后不再全模块刷新

`web_demo/core/api-client.js`

- `refreshAfterDataImport` 不再并发拉取 product / competitor / listing / traffic / report / todo。
- 改为只刷新：
  - operating-unit snapshot
  - pipeline stages

### 4. 总览页不再触发旧投影重算

`src/services/dashboard_service.py`

- 删除总览页旧投影依赖：
  - `projection_summary`
  - `projected_products`
  - `projected_report_groups`
- 总览改为读取经营快照 + 轻量任务摘要。

### 5. 清空数据清掉 V12.13 新表

`src/services/system_service.py`

清空运行态时新增清理：

- `operating_unit_snapshots`
- `pipeline_stage_gates`

避免清空测试数据后旧快照/旧阀门残留。

### 6. 任务页 GET 只读

`src/api/routes/modules/todo.py`

- `GET /api/modules/todo` 不再执行 `cluster_open_tasks()`。
- `GET /api/modules/todo` 不再执行 `auto_accept_ready_tasks()`。
- 新增显式接口：

```http
POST /api/modules/todo/lifecycle/sync
```

用于需要时执行聚合和自动接收。

## 验收点

1. 上传第二份报表后，不应再因为导入接口同步生成任务导致阻塞。
2. 导入返回里应出现 `pipelineSync`。
3. 导入返回里 `createdTaskCount` 应为 0。
4. 任务生成只通过 `/api/pipeline/data-versions/{data_version}/tasks/generate`。
5. 总览页不再调用旧投影服务。
6. 经营页和总览页都只读快照。
7. 清空数据后，快照和阀门也被清掉。
8. 任务页刷新不会自动改变任务状态。
