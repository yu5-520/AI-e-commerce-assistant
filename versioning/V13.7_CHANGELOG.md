# V13.7 Full Task Lifecycle Stations

## 目标

一次性补齐 V13.5、V13.6、V13.7 三段任务生命周期站点：

```text
V13.5：Task Acceptance / Assignment Station
V13.6：Submission + Review Station
V13.7：Recap + RAG Feedback Station
```

## 新增服务

```text
src/services/task_acceptance_assignment_station_service.py
src/services/task_submission_review_station_service.py
src/services/task_recap_rag_station_service.py
```

## 新增 API

```text
src/api/routes/task_lifecycle_stations.py
```

核心接口：

```http
GET  /api/task-lifecycle-stations/summary
POST /api/task-lifecycle-stations/acceptance/{task_id}/accept
POST /api/task-lifecycle-stations/acceptance/auto-sync
POST /api/task-lifecycle-stations/assignment/{task_id}/assign
GET  /api/task-lifecycle-stations/submission/{task_id}/evidence
POST /api/task-lifecycle-stations/submission/{task_id}/submit
POST /api/task-lifecycle-stations/review/{task_id}/review
POST /api/task-lifecycle-stations/recap/{task_id}/schedule
POST /api/task-lifecycle-stations/recap/{task_id}/complete
POST /api/task-lifecycle-stations/rag-feedback/{task_id}
```

## Station Registry 更新

内部任务生命周期线更新为：

```text
task_pool_station
→ task_acceptance_station / task_assignment_station
→ task_submission_station
→ task_review_station
→ recap_schedule_station
→ recap_complete_station
→ rag_feedback_station
```

## 边界

- 任务池站只负责入池。
- 接收/派发站只负责进入处理中或派发给运营。
- 提交站只负责证据提交与提交状态流转。
- 复核站只负责通过/退回。
- 复盘站只负责复盘排期和复盘完成。
- RAG回流站只负责把有效复盘生成经验候选。

## 当前完整闭环

```text
报表上传
→ 指标事实
→ 商品/店铺映射
→ 经营快照
→ 任务信号
→ RAG参照
→ Agent判断
→ 任务快照
→ 任务入池
→ 接收/派发
→ 运营提交
→ 总管复核
→ 系统复盘
→ RAG回流
```

## 说明

V13.7 让任务生命周期从 todo route 中抽出，成为标准站点链路。Todo 页面后续应只做任务投影和操作入口，不再作为隐藏生命周期主入口。
