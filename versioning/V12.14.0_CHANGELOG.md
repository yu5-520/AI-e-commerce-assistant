# V12.14.0 Station Contract & Ops Diagnostic Train

## 目标

把站点从流程概念升级成独立可注册、可检查、可重跑、可巡检的标准接口模块。

## 新增后端服务

- `src/services/station_registry_service.py`
- `src/services/station_contract_service.py`
- `src/services/ops_diagnostic_train_service.py`

## 新增后端接口

### Station Interface

```http
GET  /api/stations
GET  /api/stations/{station_id}
GET  /api/stations/{station_id}/contract
GET  /api/stations/{station_id}/health
GET  /api/stations/{station_id}/gates
GET  /api/stations/{station_id}/latest
POST /api/stations/{station_id}/run
POST /api/stations/{station_id}/replay
```

### Ops Diagnostic Train

```http
POST /api/ops/train/run
GET  /api/ops/train/latest
GET  /api/ops/train/runs
GET  /api/ops/train/runs/{run_id}
GET  /api/ops/stations/health
POST /api/ops/stations/{station_id}/check
```

## 验收点

1. `/api/stations` 可返回所有站点注册信息。
2. 每个站点可返回 contract / health / gates / latest。
3. `POST /api/ops/train/run` 可跑一次诊断巡检。
4. 运维火车使用 diagnostic data version，不跑真实业务数据。
5. 系统页可显示站点健康和最近巡检结果。
6. 清空演示环境会清掉 ops 巡检表。
7. 业务页面仍只读快照，不触发上游重算。

## 架构边界

V12.14 仍是模块化单体，不是物理微服务。目标是先完成逻辑隔离、接口隔离、状态隔离和巡检隔离。后续如果需要，可以把站点物理拆服务。
