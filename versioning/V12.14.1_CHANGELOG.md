# V12.14.1 Station Contract Cleanup

## 目标

把旧 Pipeline 从主流程里收口为 Station Interface 兼容层，同时隔离运维火车诊断阀门和真实业务阀门。

## 核心改动

### 1. Pipeline 变成兼容层

`src/api/routes/pipeline.py`

- `/api/pipeline/data-versions/{data_version}/operating-unit-snapshot` 转发到 `operating_snapshot_station`。
- `/api/pipeline/data-versions/{data_version}/tasks/generate` 转发到 `task_signal_station`。
- 旧 Pipeline 不再直接调用 `generate_risk_tasks_for_signals`。

### 2. 新增 Station Adapter

`src/services/station_adapter_service.py`

- 负责 Station Interface 到旧内部服务的窄桥接。
- 当前真实 adapter：
  - `operating_snapshot_station` → `materialize_operating_unit_snapshot`
  - `task_signal_station` → `generate_risk_tasks_for_signals`
- 其他站点保持 contract-only adapter，后续逐站接入。

### 3. Gate 隔离

`src/services/pipeline_gate_service.py`

新增：

- `run_type`
- `is_diagnostic`

默认 `stage_summary()` 只返回 business gate。诊断 gate 只有在：

- `includeDiagnostic=true`
- 或 `dataVersion` 为 `DIAG-*`

时才显示。

### 4. Station Contract 收口

`src/services/station_contract_service.py`

- `run_station_contract()` 统一调用 adapter。
- 业务运行写 business gate。
- 运维火车写 diagnostic gate。
- `station_gates()` 支持 `includeDiagnostic`。

### 5. 主入口清理

`src/api/main.py`

- API_VERSION 更新为 `12.14.1`。
- 旧全局 startup hook 不再在主入口执行。
- Agent/RAG 增强后续应由 `agent_enhance_station` 显式运行。

## 验收点

1. `GET /api/pipeline/stages` 默认不显示 `DIAG-*` 运维火车 gate。
2. `GET /api/pipeline/stages?includeDiagnostic=true` 可以看到诊断 gate。
3. `POST /api/pipeline/data-versions/{data_version}/tasks/generate` 返回 `compatibilityLayer = station_interface`。
4. `POST /api/stations/task_signal_station/run` 可以生成任务。
5. `POST /api/ops/train/run` 仍然只写 diagnostic gate。
6. 系统页站点巡检不污染业务 gate 计数。

## 当前边界

V12.14.1 仍是模块化单体。目标不是物理微服务，而是完成主流程调用边界：旧接口只能作为兼容层，标准站点接口才是主路。
