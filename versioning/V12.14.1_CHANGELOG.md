# V12.14.1 Station Interface Mainline Cleanup

## 目标

把旧 Pipeline 收口为 Station Interface 兼容层，同时隔离运维火车诊断阀门和真实业务阀门。

## 核心改动

### 1. Pipeline 变成兼容层

`src/api/routes/pipeline.py`

- 快照生成入口转发到 `operating_snapshot_station`。
- 任务生成入口转发到 `task_signal_station`。
- 旧 Pipeline 只保留兼容入口，主路进入 Station Interface。

### 2. Station Contract 增加窄真实 adapter

`src/services/station_contract_service.py`

当前真实 adapter：

- `operating_snapshot_station`：生成经营页快照
- `task_signal_station`：生成任务信号

其他站点保持 contract-only adapter，后续逐站接入。

### 3. Gate 隔离

`src/services/pipeline_gate_service.py`

新增：

- `run_type`
- `is_diagnostic`

默认 `stage_summary()` 只返回 business gate。诊断 gate 只有在 `includeDiagnostic=true` 或 `dataVersion` 为 `DIAG-*` 时才显示。

### 4. 主入口清理

`src/api/main.py`

- API_VERSION 更新为 `12.14.1`。
- 旧全局 startup hook 不再在主入口执行。
- Agent/RAG 增强后续应由 `agent_enhance_station` 显式运行。

## 验收点

1. `GET /api/pipeline/stages` 默认不显示 `DIAG-*` 运维火车 gate。
2. `GET /api/pipeline/stages?includeDiagnostic=true` 可以看到诊断 gate。
3. `POST /api/pipeline/data-versions/{data_version}/tasks/generate` 返回 `compatibilityLayer = station_interface`。
4. `POST /api/stations/task_signal_station/run` 可以生成任务。
5. `POST /api/ops/train/run` 仍然只写 diagnostic gate。
6. 系统页站点巡检不污染业务 gate 计数。

## 当前边界

V12.14.1 仍是模块化单体。目标不是物理微服务，而是完成主流程调用边界：旧接口作为兼容层，标准站点接口作为主路。
