# V12.11.1 链路清理版

## 目标

清理 V12.11 后仍占流程位置的旧文件和旧动作入口，确保任务链路统一为：

报表/模块任务 → V12.11 SOP任务包 → 自动接收 → 详情页 → 提交页 → 统一生命周期状态机 → 系统自动复盘。

## 更新内容

1. 提交证据链路清理
   - `task_evidence_service.submit_task_evidence()` 只保存提交材料，不再调用旧 `submit_task()`。
   - `/todo/{task_id}/submit-evidence` 先保存证据，再由统一生命周期状态机执行 `submit`。

2. 派发链路清理
   - `/todo/{task_id}/split` 和 `/todo/{task_id}/assign` 不再直接 `update_task()`。
   - 新增生命周期动作 `assign / split`，统一写状态、事件、日志、Repository 镜像和前端投影。

3. 手动建任务链路清理
   - 新增 `src/services/v1211_manual_task_package_service.py`。
   - 商品、竞品、上新、流量、报表模块的手动建任务入口统一包装为 V12.11 SOP任务包。

4. 旧文件占位清理
   - `backend/README.md` 标记为 deprecated。
   - 明确当前后端入口为 `src.api.main:app`，不再使用 `python backend/server.py`。

5. 版本同步
   - API version: `12.11.1`
   - 前端缓存: `12.11.1`
   - todo route: `12.11.1`
   - lifecycle state machine: `12.11.1`
   - action surface: `12.11.1`
   - repository write path: `12.11.1`
   - risk task facade: `12.11.1`
   - versioning/VERSION.md: `12.11.1`

## 验收点

- 任务页点击提交，进入提交页。
- 提交页确认提交后，状态进入等待系统自动复盘，不出现旧的待复核跳变。
- 商品/竞品/上新/流量/报表模块手动加入任务不再因为旧 payload 缺字段失败。
- 总管 assign/split 后，任务事件、日志、Repository、前端投影保持一致。
- ECS 不再根据 `backend/README.md` 误判旧启动入口。
