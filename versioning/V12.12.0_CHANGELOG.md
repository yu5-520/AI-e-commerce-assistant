# V12.12.0 RAG + LLM 商品级SOP增强

## 目标

把 V12.11 的规则模板式 Agent 升级为：系统变化包 + 商品上下文 + RAG基础库 + LLM生成 + SOP校验器 + 商品级动作卡。

## 更新内容

1. RAG基础库增强
   - 新增 `src/services/v1212_rag_llm_agent_service.py`。
   - 启动时写入基础 RAG 卡：
     - `metric_rule_card`
     - `action_playbook_card`
     - `sop_guardrail_card`
     - `recap_rule_card`
   - RAG卡覆盖 ROI/GMV/广告消耗、点击/转化、库存、季节服饰、系统自动复盘和禁止运营拆数据等基础规则。

2. RAG检索接入任务生成
   - 经营节奏任务在 V12.11 systemChangePack 后继续增强。
   - 生成 productContextPack。
   - 检索 RAG 经验卡和 guardrail 卡。
   - 写入 `ragRetrievedContext`。

3. LLM生成接入
   - 新增 prompt：`prompts/rag_agent_sop_generation.md`。
   - 使用统一 `llm_provider_service.generate_json(...)`。
   - LLM 未启用或输出不合格时，自动回退到 deterministic RAG synthesis。
   - 写入 `llmGeneration` 和 `llmPromptPayload`，方便后续排查。

4. SOP校验器
   - 校验字段：title、judgment、operatorSopSteps、systemRecapLine、productActionCards。
   - 禁止运营动作：拆分流量来源、拆分广告计划、人工复盘、人工判断原因、排查原因、复核数据、观察一下、持续关注。
   - 要求 SOP 包含商品、动作、提交材料、系统复盘线。

5. 商品级动作卡
   - 任务写入 `productActionCards`。
   - 批量任务写入 `affectedProducts`，保留 productId、productTitle、objectId、productLink、openProductState、primaryAction、why、submitEvidence。
   - 任务详情页新增“涉及商品”和“RAG召回依据”。
   - 商品详情页支持从任务详情跳转到具体商品档案。

6. 版本同步
   - API_VERSION: 12.12.0
   - Frontend cache: 12.12.0
   - Task cluster: 12.12.0
   - Risk task facade: 12.12.0

## 验收点

- 重新上传报表生成任务后，详情页出现“涉及商品”。
- 批量任务不再只显示“6个商品”，可看到每个商品的商品ID、商品标题、动作原因和查看商品按钮。
- 详情页出现“RAG召回依据”。
- Agent SOP 不再只依赖 if/else 模板；LLM未启用时也会基于RAG卡片生成更贴近商品的SOP。
- SOP不应要求运营拆分数据或人工复盘。
