# V13.3 Task Snapshot Station

## 目标

新增任务快照站，把 Agent 判断结果先沉淀为标准任务快照，再由后续任务池站决定是否入池。

## 核心原则

任务不是系统规则直接创建的。系统给事实，RAG给参照，Agent给判断，任务快照记录判断结论。

## 当前边界

V13.3 不直接创建任务池任务。任务快照只说明“是否应该进入任务池、为什么进入、证据是什么、风险是什么、Agent判断依据是什么”。

## 后续版本

V13.4 应把 `snapshot_ready` / `manager_review_required` 的任务快照送入 `task_pool_station`，再进入任务生命周期。
