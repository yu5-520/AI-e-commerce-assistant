# V12.13.0 Pipeline Station Interface & Gate

## 目标

把主流程从“页面触发全链路重算”改成“货运火车式分站接力”。

## 核心原则

- 主流程不循环。
- 每个小流程是独立站点。
- 每站只接收上一站标准产物。
- 每站完成后写入阀门。
- 页面只读取快照和任务包。
- 真正能循环学习的只有 RAG 回流与增强。

## 关键改动

### 1. 经营页快照读取

`GET /api/modules/operating-unit` 现在只读取 `operating_unit_snapshot`。

它不再执行：

- `projection_summary`
- `projected_products`
- `projected_traffic`
- `dataset_rows`
- RAG 检索
- LLM 生成
- 任务生成

### 2. 显式快照重建

新增：

```http
POST /api/modules/operating-unit/snapshot/rebuild
```

用于手动重建经营页快照。

### 3. 阶段阀门

新增 `pipeline_stage_gates` 表，记录：

- tenant_id
- user_id
- data_version
- stage
- status
- input_hash
- output_hash
- upstream_stage
- output_ref
- error_message
- started_at
- finished_at

### 4. Pipeline接口

新增：

```http
GET  /api/pipeline/stages
POST /api/pipeline/data-versions/{data_version}/stage/{stage}/complete
POST /api/pipeline/data-versions/{data_version}/operating-unit-snapshot
POST /api/pipeline/data-versions/{data_version}/tasks/generate
```

### 5. 任务生成独立站点

任务生成只通过显式 pipeline 接口触发，不再由经营页刷新触发。

## 验收点

- 上传第二份报表后，经营页不应再 504。
- 经营页响应中应出现 `readMode = snapshot_only`。
- 响应中应出现 `forbiddenRuntimeStages`。
- `pipelineGate` 中能看到 `operating_unit_snapshot_ready`。
- 任务生成只通过 `/api/pipeline/data-versions/{data_version}/tasks/generate` 触发。
- RAG 回流仍是唯一学习循环。
