# V12.14.2 Deprecated Station Archive & Mainline Purity

## 目标

新增废弃站点 / 存储站点，把旧文件、旧接口、旧补丁、旧文档从主架构里隔离出去，保持业务正线站点干净。

## 核心改动

### 1. 新增废弃站点注册表

`src/services/deprecated_station_registry_service.py`

登记：

- 旧文件
- 旧 Hook
- 旧兼容路由
- legacy adapter 依赖
- 替代站点
- 允许用途
- 风险等级
- 删除计划

### 2. 新增废弃站点接口

`src/api/routes/deprecated_stations.py`

```http
GET /api/deprecated-stations
GET /api/deprecated-stations/risks
GET /api/deprecated-stations/mainline-check
GET /api/deprecated-stations/{legacy_id}
```

### 3. 主 Station Registry 清理

`agent_enhance_station` 不再直接指向：

```text
src.services.v1212_rag_llm_agent_service
```

改为干净站点模块：

```text
src.stations.agent_enhance_station.service
```

旧 V12.12 文件进入 Deprecated Station Archive。

### 4. 新增干净 Agent Enhance Station 壳

新增：

```text
src/stations/agent_enhance_station/service.py
```

当前只作为 clean station shell，真实 RAG/LLM 增强后续迁移到本站 adapter。

### 5. 运维火车增加旧文件回流检查

Ops Diagnostic Train 在巡检站点前先检查：

```text
deprecated_mainline_leak
```

检查内容：

- main.py 是否执行旧 Hook
- station_registry 是否直接引用 archive_only 旧文件
- pipeline.py 是否直接 import 旧业务服务
- frontend entry 是否加载 deprecated 文件

### 6. 新增架构文档

新增：

```text
docs/architecture/main_station_architecture.md
docs/architecture/deprecated_station_archive.md
```

主架构文档只写业务正线，废弃文档只写旧文件治理。

## 验收点

1. `/api/deprecated-stations` 可返回旧文件治理清单。
2. `/api/deprecated-stations/mainline-check` 可检查旧文件是否回流主路。
3. `agent_enhance_station` 不再指向 V12.12 旧文件。
4. 运维火车结果包含 `deprecated_mainline_leak` 检查。
5. 主架构文档和废弃文档分离。

## 当前边界

V12.14.2 先做登记和隔离，不大规模物理移动旧文件，避免 import 断点。后续版本再逐步把 archive-only 文件迁移或删除。
