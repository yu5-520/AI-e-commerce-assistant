# V12.14.3 Deprecated Physical Archive Migration

## 目标

执行第一批 archive_only 旧文件的物理归档：旧补丁文件不再继续停留在 `src/services` 主服务目录，而是进入 `src/deprecated_stations/archive_services`。

## 新增目录

```text
src/deprecated_stations/
src/deprecated_stations/archive_services/
```

## 第一批归档对象

```text
src/services/v112_task_chain_fix_service.py
src/services/v1211_agent_sop_enhancement_service.py
src/services/v1212_rag_llm_agent_service.py
```

对应 archive 记录：

```text
src/deprecated_stations/archive_services/v112_task_chain_fix_service.py
src/deprecated_stations/archive_services/v1211_agent_sop_enhancement_service.py
src/deprecated_stations/archive_services/v1212_rag_llm_agent_service.py
```

## Registry 更新

`src/services/deprecated_station_registry_service.py` 更新为 `12.14.3`：

- archive_only 文件路径指向新 archive 目录
- 增加 `originalPath`
- `mainline_purity_check()` 会检查 archive_only 原路径是否重新出现

## 当前边界

这次只迁移 archive_only 的旧补丁。以下 legacy adapter 依赖暂不移动：

```text
src/services/risk_task_service.py
src/services/operating_unit_snapshot_service.py
src/services/metric_fact_store_service.py
```

这些仍由 Station Adapter 保护，不能直接被旧 route 或前端调用。

## 验收点

1. `API_VERSION = 12.14.3`
2. `/api/deprecated-stations` 返回 archive 文件的新路径
3. `/api/deprecated-stations/mainline-check` 不应出现 archive 原路径残留
4. `src/services` 不再保留第一批 archive_only 旧补丁
5. `python -m compileall src` 可通过
