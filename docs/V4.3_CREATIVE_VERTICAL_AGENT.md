# V4.3 Creative Vertical Agent

V4.3 adds the vertical category creative Agent.

The product rule is:

```text
标题主图 Agent 不是素材生成器，也不是让运营继续想标题。
它是测试包生成器：Agent 生成可复制上架的标题 / 主图 / 卖点组合，运营负责选择、上架测试、反馈结果。
```

## 1. Flow

```text
product facts
↓
category profile
↓
platform expression rules
↓
competitor signals
↓
RAG creative / playbook memory
↓
test packages
↓
operator chooses package
↓
operator lists / tests package
↓
operator submits metrics
↓
manager reviews result
↓
feedback flywheel writes experience-card draft
```

## 2. API

```text
POST /api/modules/agents/creative/{product_id}
GET  /api/modules/agents/creative/{product_id}
POST /api/modules/agents/creative/{product_id}/tasks
```

`POST /api/modules/agents/creative/{product_id}/tasks` accepts:

```json
{
  "packageIndex": 0,
  "taskGoal": "提升点击率",
  "platform": "拼多多",
  "categoryId": "home_living_goods"
}
```

## 5. Platform rules

```text
淘宝：搜索关键词覆盖、材质、场景、规格可信度。
拼多多：到手价值、套装数量、价格带、耐用。
抖音小店：场景痛点、短句卖点、人群需求、即时改善。
通用：商品事实、核心卖点、使用场景、风险边界。
```

## 6. Safety boundary

The creative Agent can:

```text
generate title variants
generate main-image directions
rank selling points
generate ready-to-test packages
create a selected-package test task draft
cite RAG references
```

It cannot:

```text
directly publish products
directly change price
directly change ad spend
directly write to ERP / CRM / marketplace data
generate exaggerated claims
skip human confirmation
```

## 7. Operator role

```text
运营不是从零想标题和主图。
运营选择 Agent 生成的测试包，上架小流量测试，提交数据和胜出方案。
```
