# DATA_TASK_LIFECYCLE

本文件只记录当前产品从数据到任务的主链路。后续 AI 修改任务系统、报表系统、经营模块时，以本文件为定位依据。

## 1. 主生命周期

```text
报表导入
→ 字段映射
→ 数据校验
→ DataVersion
→ 原始行入库 / 运行态记录
→ 模块投影
→ 趋势计算
→ 风险信号
→ Agent 经营档案
→ 自动标签
→ 标签变化任务
→ 统一任务池
→ 跨账号流转
→ 运营接收
→ 运营提交证据
→ 总管复核
→ 老板查看结果
→ 日志留痕
→ RAG 记忆候选
```

## 2. 导入阶段

入口：

```text
web_demo/modules/report/page.js
→ AppApi.confirmReportImport()
→ /api/data/import/confirm
→ src/api/routes/data_import.py
```

导入阶段必须完成：

- 识别数据集。
- 完成字段映射。
- 创建或更新数据版本。
- 触发模块刷新契约。
- 触发趋势和风险任务同步。
- 返回前端可刷新目标。

## 3. 投影阶段

数据导入后，系统需要把报表行转成经营模块可使用的视图：

- 商品卡片。
- 竞品卡片。
- 上新卡片。
- 流量卡片。
- 库存和售后状态。
- 总览任务台摘要。

任何经营模块都不应该直接读取前端临时状态作为判断依据。

## 4. 趋势和风险阶段

趋势不是单点指标，必须支持变化判断：

- 环比。
- 同比。
- 趋势方向。
- 波动幅度。
- 指标联动。
- 数据版本对比。

风险信号进入任务前，必须能说明：

- 哪个商品或店铺触发。
- 来自哪个数据版本。
- 哪个指标异常。
- 为什么需要任务介入。

## 5. 任务生成阶段

任务必须包含：

- task id。
- 来源模块。
- 来源实体。
- 风险域。
- 任务标题。
- 处理动作。
- 优先级。
- 截止时间。
- 判断标签。
- 证据要求。
- 数据版本。

任务进入统一任务池后，其他模块不再重复展示已完成任务。

## 6. 跨账号流转阶段

```text
总管派发
→ 运营接收
→ 运营提交
→ 总管复核
→ 老板查看
→ 日志 / RAG 候选
```

同一个 task id 在不同角色下展示不同视图：

- 老板：查看进度和结果。
- 总管：派发、复核、驳回。
- 运营：接收、提交、补充。

## 7. 完成和留痕阶段

任务完成后必须：

- 从待处理任务池中移出。
- 同步更新来源模块展示状态。
- 写入任务事件。
- 写入日志。
- 生成可复盘内容。
- 必要时生成 RAG 记忆候选。

## 8. 验收标准

一次完整验收必须覆盖：

```text
导入测试数据
→ 总览刷新
→ 经营模块刷新
→ 生成任务
→ 三端角色可见
→ 运营接收提交
→ 总管复核通过
→ 任务完成
→ 日志留痕
→ RAG 候选生成
```
