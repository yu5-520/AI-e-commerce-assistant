# 简历项目表述

## 项目标题

AI + RPA + ERP + CRM 电商经营自动化工作台 MVP

## 简历短版

围绕电商 ERP / CRM 经营场景，设计并搭建 Workflow-first 的 AI 自动化工作台 MVP，使用 Mock 商品、订单、库存、退款、客户标签和互动数据，跑通数据导入、商品诊断、客户分层、RAG 召回、RPA 任务草案、人工确认和报告输出闭环。

## 简历 Bullet

- 设计 Workflow-first AI 工作流架构，将 ERP 商品 / 订单 / 库存 / 退款数据与 CRM 客户 / 标签 / 互动数据统一进入商品档案和客户档案。
- 构建 Mock Workflow，跑通“数据导入 → 商品诊断 → 客户分层 → RAG 召回 → RPA 任务草案 → 人工确认 → 报告输出”的最小闭环。
- 设计 RAG 知识增强骨架，拆分平台规则、合规风控、运营方法和客服 SOP 知识源，用于支持商品诊断、售后归因和客户触达风险判断。
- 设计 Human-in-the-loop 机制，将改价、投放、活动报名、客户触达和退款处理等高风险动作置于人工确认之后，RPA 仅生成低风险任务草案。
- 新增 Evals 评测样例，验证 CRM 是否识别售后敏感客户、RPA 是否禁止自动改价 / 群发 / 退款、任务是否默认进入人工确认。
- 搭建静态前端 Demo，展示导入 Mock 数据、AI / RAG 诊断、RPA 任务草案和人工确认项。

## 面试解释短句

当前项目不是生产级系统，而是可运行 MVP 骨架。第一阶段使用规则引擎模拟 AI 诊断、关键词检索模拟 RAG、结构化 JSON 模拟 RPA 任务草案，先验证业务流程和安全边界；后续可逐步替换为真实 LLM、Embedding 向量库、ERP / CRM API 和 RPA 执行器。
