# V4.1 RAG Memory Flywheel

V4.1 turns the module Agent runtime into an operation experience flywheel.

The product rule is:

```text
日志不是知识库。日志必须先被提炼为经验卡，经过质量分和人工复核后，才能进入 RAG 召回。
```

## 1. Flywheel

```text
ERP / CRM / report data
↓
warning and task generation
↓
module Agent advice
↓
operator handling
↓
manager review
↓
daily / weekly / task logs
↓
feedback memory agent
↓
structured experience card
↓
owner / manager approval
↓
RAG memory retrieval
↓
next task generation and task interpretation
```

## 2. Memory levels

```text
L0 原始日志 / 拒绝入库
L1 经验草案
L2 已复核经验
L3 高质量经验
L4 失败案例 / 避坑边界
```

## 3. API

```text
GET  /api/modules/rag-memory
GET  /api/modules/rag-memory/cases
GET  /api/modules/rag-memory/search
POST /api/modules/rag-memory/feedback/tasks/{task_id}
POST /api/modules/rag-memory/cases/{case_id}/approve
POST /api/modules/rag-memory/cases/{case_id}/reject
```

## 4. Experience card contract

Each experience card is structured instead of being raw text:

```text
caseId
caseType
level
status
categoryId
platform
storeId
problemType
operatorStyle
title
initialJudgment
effectiveActions
applicableConditions
notApplicableConditions
resultSummary
beforeMetrics
afterMetrics
qualityScore
effective
reviewStatus
sourceTaskId
validUntil
```

## 5. Quality gate

An experience card should not be used for formal RAG retrieval unless it passes the memory gate:

```text
manager or owner review
clear handling action
before / after metric evidence
applicable conditions
not-applicable conditions
qualityScore >= 0.7
```

## 6. Current retrieval mode

V4.1 uses a lightweight retrieval layer:

```text
SQLite structured cards
+ category / platform / store / problem filters
+ quality score
+ simple keyword scoring
```

Production can later replace the retrieval internals with embeddings and rerank without changing the API contract.

## 7. Safety boundary

RAG memory can:

```text
recall approved playbooks
recall historical cases
recall negative cases as boundaries
support task generation
support task interpretation
support title / main-image strategy
```

RAG memory cannot:

```text
auto-write raw logs into formal memory
auto-approve operator claims
directly change price
directly change ad spend
directly refund
directly publish or delist products
directly write to real ERP / CRM / marketplace data
```
