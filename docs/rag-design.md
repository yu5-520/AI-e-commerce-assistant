# RAG 知识库架构设计

## 1. 设计目标

RAG 的目标不是堆大量资料，而是解决电商 AI 运营决策中的三个问题：

1. AI 单次生成缺少商品历史上下文。
2. AI 对平台规则、合规边界和活动要求缺少稳定依据。
3. AI 输出容易出现真实性偏差，无法追溯依据。

因此，本项目采用分层知识库设计，让 AI 在生成商品诊断、SKU 建议、活动准备和 RPA 任务草案前，先检索相关知识，再注入 Prompt 上下文。

## 2. RAG 总体流程

```text
知识文档导入
↓
数据清洗
↓
文档切片 Chunking
↓
Embedding 向量化
↓
向量库索引
↓
业务任务触发
↓
检索 Top-K 相关知识
↓
Prompt 上下文注入
↓
AI 输出策略 / 检查表 / RPA 任务草案
↓
Evals 评测与召回日志记录
```

## 3. 知识库分层

### 3.1 商品档案库

用于保存单个商品的长期运营记忆。

内容包括：

- 商品基础资料
- 历史标题版本
- 历史主图方案
- SKU 方案
- 测试记录
- 订单与退款摘要
- 活动记录
- AI 复盘报告
- 用户确认记录
- RPA 执行记录

作用：

- 避免每次重新生成。
- 让 AI 基于上一轮数据判断下一轮动作。
- 提升复盘连续性和可追溯性。

### 3.2 平台规则库

用于保存平台运营规则。

内容包括：

- 类目规则
- 活动规则
- 广告规则
- 商品发布规则
- 标题与主图要求
- 资质要求
- 售后与退款规则

作用：

- 在活动报名、标题生成、主图建议时提供规则依据。
- 避免 AI 给出与平台规则冲突的建议。

### 3.3 合规风控库

用于保存高风险表达和禁用边界。

内容包括：

- 敏感词
- 违禁表达
- 高风险类目
- 虚假宣传风险
- 侵权素材风险
- 未授权品牌词风险
- 自动化操作边界

作用：

- 在生成运营材料前先做风险过滤。
- 对高风险商品优先输出合规检查，而不是增长建议。

### 3.4 运营方法库

用于保存可复用的运营方法论。

内容包括：

- 标题测试方法
- 主图测试方法
- SKU 利润拆分
- 库存清货策略
- 活动报名准备
- 半付费测试方法
- 运营日报模板
- 复盘分析框架

作用：

- 让 AI 输出不是单纯文案，而是带有运营逻辑的方案。
- 支持 RPA 生成结构化日报、活动表和任务清单。

## 4. 文档切片策略

MVP 阶段建议采用规则切片：

| 知识类型 | 切片方式 | 示例 |
|---|---|---|
| 商品档案 | 按商品 + 测试轮次 | P001 第 1 轮标题测试 |
| 平台规则 | 按规则主题 | 活动报名价格规则 |
| 合规风控 | 按风险类型 | 医药功效表达风险 |
| 运营方法 | 按任务类型 | SKU 利润测算方法 |

每个 chunk 建议包含：

```json
{
  "chunk_id": "rule_activity_price_001",
  "source_type": "platform_rule",
  "title": "活动价不得低于保本线",
  "content": "活动价需要结合成本、物流、退款损耗和平台费用判断，不建议低于保本价参与活动。",
  "tags": ["活动", "价格", "利润风险"],
  "risk_level": "medium"
}
```

## 5. 检索策略

根据业务任务选择检索范围：

| 业务任务 | 优先检索知识库 |
|---|---|
| 商品诊断 | 商品档案库 + 运营方法库 |
| SKU 利润测算 | 商品档案库 + 运营方法库 |
| 活动报名准备 | 平台规则库 + 商品档案库 + 合规风控库 |
| 标题生成 | 商品档案库 + 平台规则库 + 合规风控库 |
| 主图方案 | 商品档案库 + 合规风控库 + 运营方法库 |
| RPA 任务生成 | 商品档案库 + 人工确认规则 + 操作边界 |

## 6. Prompt 上下文注入格式

AI 生成前注入以下上下文：

```text
【商品档案摘要】
...

【ERP / 经营数据摘要】
...

【召回的平台规则】
...

【召回的合规风险】
...

【召回的运营方法】
...

【任务要求】
请基于以上内容输出结构化运营建议，不要编造缺失数据，涉及高风险动作必须提示人工确认。
```

## 7. RAG Evals 评测

评测维度：

- 是否召回正确商品历史记录
- 是否召回相关平台规则
- 是否识别合规风险
- 是否避免编造数据
- 是否能说明建议依据
- 是否将高风险动作交给人工确认

## 9. 设计结论

RAG 在本项目中的价值不是“让 AI 知道更多”，而是：

> 让 AI 的每一次运营判断都有商品历史、平台规则、合规边界和运营方法作为依据，从而降低真实性偏差，并提升结果的可追溯性。
