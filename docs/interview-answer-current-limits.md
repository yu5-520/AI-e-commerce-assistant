# 面试问答：当前项目边界和不足

## 问题

这个项目现在做到什么程度？还有哪些没有完成？

## 回答

当前项目已经完成了 Workflow-first 架构设计、Mock ERP / CRM 数据、规则诊断节点、简单 RAG 检索、RPA 任务草案、Evals 评测和静态前端 Demo。

可以跑通：

```text
Mock 数据导入
↓
商品诊断
↓
客户分层
↓
RAG 召回
↓
RPA 任务草案
↓
人工确认边界
↓
报告输出
```

但当前还不是生产级系统。

尚未完成：

- 尚未接入真实 ERP / CRM API。
- 尚未接入真实店铺后台。
- 尚未接入真实大模型 API。
- RAG 目前是关键词检索，尚未接 Embedding 和向量库。
- 前端目前是静态 Demo，尚未通过 API 调用 Python 工作流。
- 任务确认 / 拒绝状态流转还没有做成真实后端记录。

## 一句话总结

> 当前项目是一个可运行的 MVP 骨架，重点验证业务流程、数据结构、AI 工作流和安全边界，后续可以逐步替换为真实模型、真实 RAG、真实 API 和真实 RPA 执行器。
