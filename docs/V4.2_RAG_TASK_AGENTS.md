# V4.2 RAG Task Agents

V4.2 adds two Agents on top of the V4.1 structured experience memory layer:

```text
自动解析生成任务 Agent
任务解析运营方式 Agent
```

The product rule is:

```text
规则负责稳定触发，RAG 负责召回经验，Agent 负责生成任务草案和打法解释，人审负责是否进入任务池和是否执行。
```

## 1. Task generation flow

```text
module data / report signal / metric abnormality
↓
rule hits
↓
problem type classification
↓
RAG memory retrieval
↓
confidence scoring
↓
task candidate
↓
human confirmation or optional high-confidence task creation
↓
unified task pool
```

## 2. Playbook flow

```text
active task
↓
problem type
↓
RAG playbook / historical case / negative case retrieval
↓
multiple operating styles
↓
operator chooses style
↓
evidence submission
↓
manager review
↓
feedback memory card
```

## 6. Operating styles

V4.2 starts with three operating styles:

```text
稳健型：先控风险，补齐证据，再逐步恢复流量或动作。
增长型：保留小额测试，快速换素材 / 方向，但严控失败阈值。
利润型：优先核算真实毛利、退款损耗和活动承接。
```

The Agent does not force one answer. It shows multiple paths so the operator and manager can choose based on current goals.

## 7. Safety boundary

The task Agents can:

```text
generate task candidates
generate task drafts
generate evidence requirements
generate operating playbooks
cite RAG references
suggest human decision points
```

They cannot:

```text
directly change price
directly change ad spend
directly refund
directly publish or delist products
directly write to real ERP / CRM / marketplace data
skip manager review
skip RAG memory approval
```
