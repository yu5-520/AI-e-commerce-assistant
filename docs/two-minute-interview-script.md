# 2 分钟面试讲解稿

## 第一段：项目背景

我这个项目最早是从电商 AI 商品增长助手切入的，但后来我发现真实招聘市场里，企业更关心的不是单次生成标题或主图，而是 AI 能不能接入 ERP、CRM 和 RPA 这些真实业务流程。

所以我把项目升级成了 AI + RPA + ERP + CRM 电商经营自动化工作台 MVP。

## 第二段：项目架构

项目采用 Workflow-first 架构，而不是深度自治 Agent。

主流程是：

```text
ERP / CRM 数据导入
↓
商品档案与客户档案
↓
RAG 召回平台规则、合规边界、运营方法和客服 SOP
↓
AI 做商品诊断、客户分层和售后归因
↓
生成 RPA 任务草案
↓
人工确认高风险动作
↓
日志回写和复盘
```

## 第三段：当前实现

目前已经实现了一个可运行的 Mock Workflow。

可以通过：

```bash
python -m src.run_demo
```

生成商品诊断、客户分层、RPA 任务草案、RAG 召回上下文和 Markdown 报告。

也有一个静态前端 Demo，可以展示导入数据、AI 诊断、RPA 任务草案和人工确认项。

## 第四段：安全边界

这个项目没有让 AI 自动改价、自动投放、自动报名活动或自动触达客户。

所有高风险动作都必须人工确认。

这也是我选择 Workflow-first 的原因：电商 ERP / CRM 场景更适合可控流程，而不是一开始就做无边界 Agent。

## 第五段：下一步

下一步我会用 FastAPI 把 Python Workflow 封装成接口，再让前端通过 fetch 调用真实后端结果，并增加任务确认 / 拒绝 / 日志回写。

这样项目就能从静态展示升级成前后端可交互 Demo。
