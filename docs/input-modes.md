# 输入端模式设计

本仓库采用三模式输入，不再使用大而全表单。

## 原则

用户先选模式，再填最少信息。

这样可以减少 AI 自动选路风险，也减少无关字段。

## 模式

1. 自然流测试
- 模板：`.github/ISSUE_TEMPLATE/01-natural-flow.yml`
- 标题：`[自然流]`
- 输入：商品、当前问题、想测标题/主图/价格/SKU、已有信息。

2. 强付费放量
- 模板：`.github/ISSUE_TEMPLATE/02-paid-growth.yml`
- 标题：`[强付费]`
- 输入：商品、预算、成本、库存、当前数据、投放问题。

3. 爆品打造
- 模板：`.github/ISSUE_TEMPLATE/03-hot-product.yml`
- 标题：`[爆品]`
- 输入：自家商品、参考爆品、路线、备货能力、当前数据、目标。

## 输出

Action 根据标题前缀生成对应结果卡。
