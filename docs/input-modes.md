# 输入端模式设计

本仓库不再使用一个大而全的商品表单。

用户先选择运营模式，再填写该模式最少需要的信息。

这样可以：

- 减少 AI 自动判断路线带来的不稳定性
- 减少输入端冗余字段
- 降低用户启动成本
- 让输出结果更稳定

## 1. 自然流测试

模板：`.github/ISSUE_TEMPLATE/01-natural-flow.yml`

标题前缀：`[自然流]`

适合：白牌、小商家、低预算、想先测标题、主图、价格和 SKU 的商品。

输入重点：商品一句话、当前问题、想测试什么、已有标题/主图/价格/竞品。

## 2. 强付费放量

模板：`.github/ISSUE_TEMPLATE/02-paid-growth.yml`

标题前缀：`[强付费]`

适合：已有预算、库存、基础转化，准备判断能不能投放放量的商品。

输入重点：商品一句话、预算范围、售价/
