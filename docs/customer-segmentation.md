# 客户分层设计

## 1. 设计目标

客户分层用于帮助电商团队从“所有客户一视同仁”升级为“按客户价值、复购周期、售后风险和活动敏感度进行运营”。

本项目只生成客户分层报告和运营任务草案，不自动触达真实客户。

## 2. RFM 分层

RFM 是客户分层的基础方法：

- R：Recency，最近一次购买时间
- F：Frequency，购买频次
- M：Monetary，消费金额

## 3. 基础分层规则

| 客户类型 | 判断依据 | 建议动作 |
|---|---|---|
| 高价值客户 | 购买频次高、金额高、近期活跃 | 老客维护、会员权益、优先新品推荐 |
| 新客 | 首次购买时间较近 | 新客关怀、使用说明、复购引导 |
| 复购潜力客户 | 有多次购买或同类商品复购可能 | 生成复购提醒任务 |
| 沉睡客户 | 长时间未购买 | 召回策略建议，避免频繁打扰 |
| 流失风险客户 | 历史消费高但近期无互动 | 生成流失预警清单 |
| 价格敏感客户 | 多在活动期购买 | 优惠券策略草案 |
| 售后敏感客户 | 退款或投诉次数较高 | 售后归因，谨慎触达 |
| 活动敏感客户 | 活动订单占比较高 | 活动前触达草案，需人工确认 |

## 4. AI 分层输入

```text
客户基础数据
+ 历史订单
+ 退款记录
+ 最近互动时间
+ 商品类目
+ 活动参与记录
+ 客服 / 售后摘要
```

## 5. AI 分层输出格式

```json
{
  "customer_id": "C001",
  "segment": "高价值客户",
  "rfm_score": "R5F5M5",
  "tags": ["高价值", "复购潜力"],
  "basis": ["近30天有购买", "累计订单数高", "退款次数低"],
  "recommended_action": "生成老客复购任务草案",
  "risk_notes": ["不自动触达客户，需人工确认"],
  "requires_human_approval": true
}
```

## 6. 售后敏感客户识别

售后敏感客户需要谨慎处理。

可能信号：

- 多次退款
- 投诉记录
- 售后反馈负面
- 对商品描述不满
- 对物流或客服响应不满

建议动作：

- 优先生成售后归因报告
- 检查商品详情页和主图是否存在误导
- 优化客服 SOP
- 不直接推送营销内容

## 7. 分层与 RPA 结合

RPA 可执行低风险任务：

- 导出高价值客户清单
- 导出沉睡客户清单
- 生成复购任务表
- 生成售后归因表
- 生成优惠券策略建议表
- 生成客服话术草案

RPA 不执行：

- 自动群发
- 自动外呼
- 自动承诺优惠
- 自动处理退款
- 自动诱导好评

## 8. 分层与 RAG 结合

RAG 可召回：

- 客户运营方法
- 客服 SOP
- 合规触达规则
- 历史互动摘要
- 商品售后记录

## 9. 设计结论

客户分层的价值不是“精准骚扰客户”，而是：

> 帮助运营人员识别不同客户状态，生成更谨慎、更可确认、更可追溯的客户运营任务草案。
