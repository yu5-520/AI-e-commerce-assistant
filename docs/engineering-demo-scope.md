# 工程 Demo 范围说明

## 1. 当前 Demo 范围

当前 Demo 只验证业务流程和数据结构，不追求生产级实现。

已覆盖：

- Mock 数据导入
- 商品诊断
- 客户分层
- RAG 关键词检索
- RPA 任务草案
- 人工确认边界
- JSON / Markdown 输出
- 静态前端展示
- Evals 最小评测

## 2. 不在当前范围

当前不包含：

- 真实 ERP API
- 真实 CRM API
- 真实店铺后台
- 真实 RPA 执行器
- 真实 LLM API
- 向量数据库
- 用户登录和权限系统
- 生产级日志系统

## 3. 为什么这样设计

MVP 阶段的目标是先验证：

```text
业务链路是否成立
输入输出是否清楚
风险边界是否明确
Demo 是否可展示
```

而不是直接进入高成本、高风险的生产级接入。

## 4. 工程升级顺序

```text
1. FastAPI 封装当前 Workflow
2. 前端调用 API
3. 增加任务确认 / 拒绝状态
4. 接 SQLite 或轻量日志存储
5. 接 LLM API
6. 接 Embedding + 向量库
7. 接真实 ERP / CRM Adapter
8. 接 RPA 执行器
```

## 5. 结论

当前工程 Demo 是一个可解释、可演示、可替换的 AI 工作流骨架。
