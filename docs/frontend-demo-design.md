# 前端三段式 Demo 设计

## 1. 设计目标

前端 Demo 用于把仓库中的 Workflow-first 架构可视化展示出来。

它不是完整生产系统页面，而是面试和演示用的轻量原型，重点展示：

```text
导入 ERP / CRM Mock 数据
↓
生成 AI / RAG 诊断
↓
生成 RPA 任务草案
↓
查看人工确认项
```

## 2. 页面结构

当前静态页面位于：

```text
web_demo/index.html
web_demo/styles.css
web_demo/app.js
```

## 3. 三段式交互

### 3.1 第一段：数据导入

展示：

- 商品表数量
- 订单表数量
- 库存表数量
- 退款表数量
- 客户表数量
- 客户标签数量
- 互动记录数量

目标：让用户理解系统不是单次 Prompt，而是基于 ERP / CRM 数据运行。

### 3.2 第二段：AI 诊断

展示：

- 商品风险
- SKU 价格风险
- 库存风险
- 敏感类目风险
- 客户分层
- 售后敏感客户
- RAG 召回依据

目标：让用户理解 AI 是流程中的判断节点。

### 3.3 第三段：RPA 任务草案

展示：

- 运营日报任务
- SKU 价格建议表任务
- 售后归因任务
- 客户分层任务
- 人工确认状态
- 是否允许自动执行

目标：让用户理解 RPA 不是无边界执行，而是低风险、可确认、可追溯的任务草案。

## 4. 安全边界展示

页面必须明确展示：

- 不自动改价
- 不自动投放
- 不自动报名活动
- 不自动群发客户
- 不自动处理退款
- 不保存真实客户隐私

## 5. 后续升级方向

当前是静态 Mock 页面，下一步可以升级为：

```text
前端上传 CSV
↓
调用 Python / API 工作流
↓
返回真实 JSON 诊断结果
↓
页面渲染 RPA 任务草案
↓
用户点击确认 / 拒绝 / 修改
↓
日志回写
```

## 6. 设计结论

前端 Demo 的核心价值是把抽象架构变成可感知流程：

> 用户看到的不是“AI 生成文案”，而是“数据进入系统，AI 做判断，RAG 给依据，RPA 出任务，人工控风险”。
