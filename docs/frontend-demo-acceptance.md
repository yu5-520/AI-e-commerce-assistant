# 前端 Demo 验收标准

## 1. 基础验收

- [ ] 可以直接用浏览器打开 `web_demo/index.html`。
- [ ] 页面标题清楚表达 AI + RPA + ERP + CRM 电商经营自动化工作台。
- [ ] 页面能展示 Workflow-first · Agent-ready 定位。
- [ ] 页面在手机和平板宽度下不会明显错位。

## 2. 流程验收

- [ ] 点击“导入 Mock 数据”后，能展示 ERP 数据摘要。
- [ ] 点击“导入 Mock 数据”后，能展示 CRM 数据摘要。
- [ ] 点击“生成 AI 诊断”后，能展示商品诊断、客户分层和 RAG 依据。
- [ ] 点击“生成 RPA 任务草案”后，能展示任务类型、风险等级、审批状态和是否允许自动执行。
- [ ] 点击“查看人工确认项”后，能突出中高风险任务。

## 3. 安全边界验收

- [ ] 页面明确说明不自动改价。
- [ ] 页面明确说明不自动投放。
- [ ] 页面明确说明不自动报名活动。
- [ ] 页面明确说明不自动群发客户。
- [ ] 页面明确说明不自动处理退款。
- [ ] 页面明确说明不保存真实客户隐私。

## 4. 展示验收

- [ ] 面试展示时能在 2 分钟内讲清楚项目链路。
- [ ] 页面能让非技术人员看懂“数据进入系统，AI 做判断，RPA 出任务，人工控风险”。
- [ ] 页面不会让人误解为已经接入真实店铺后台或真实客户数据。

## 5. 后续 API 化验收

- [ ] 前端可以调用 `/api/demo/run`。
- [ ] 前端可以渲染真实 Python 工作流返回结果。
- [ ] 前端支持任务确认 / 拒绝状态变化。
- [ ] 前端能展示日志回写结果。
