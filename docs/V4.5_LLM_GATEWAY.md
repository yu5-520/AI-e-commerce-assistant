# V4.5 LLM Gateway

V4.5 adds the unified LLM interface layer.

The product rule is:

```text
LLM 只做表达增强和草案生成。
ActionPlan、账号权限、任务池、人审、复核和审计链路仍然是确定性系统。
```

## 1. Why not all-in MCP now

MCP is not the primary model interface in V4.5. The product first needs a stable model gateway and a safe internal tool gateway.

```text
LLM Provider Gateway：统一模型调用
Tool Gateway：统一内部安全工具
MCP Adapter：未来外部工具生态适配层
```

MCP must not bypass:

```text
/api/accounts
统一任务池
ActionPlan
人工确认
总管复核
RAG 经验卡复核入库
审计日志
```

## 2. Runtime structure

```text
Agent
↓
problemType / ActionPlan / RAG references / data snapshot
↓
LLM Gateway
├── Provider config
├── Prompt template
├── Guardrail
├── Trace
└── Fallback
↓
OpenAI-compatible provider or deterministic mock output
```

Tool structure:

```text
Agent / LLM Prompt
↓
Tool Gateway
├── get_product_snapshot
├── get_traffic_snapshot
├── get_competitor_snapshot
├── get_listing_snapshot
├── get_task_snapshot
├── search_rag_memory
└── build_action_plan
```

Blocked tools:

```text
change_price
publish_product
increase_ad_budget
refund_order
write_erp
write_crm
```

## 5. Environment

```text
LLM_ENABLED=false
LLM_MOCK_MODE=false
LLM_TRACE_ENABLED=true
LLM_PROVIDER=deepseek
LLM_MODEL=deepseek-chat
LLM_BASE_URL=
LLM_API_KEY=
LLM_TEMPERATURE=0.3
LLM_TIMEOUT=60
LLM_MAX_TOKENS=1200
```

When `LLM_ENABLED=false`, the product still runs with deterministic ActionPlan and mock LLM fallback.

## 6. Provider config

`config/model_providers.json` stores OpenAI-compatible provider metadata.

Supported style:

```text
provider name
base url
api key env
model
```

The gateway can use either generic `LLM_API_KEY` or provider-specific keys such as `OPENAI_API_KEY` / `DEEPSEEK_API_KEY`.

## 7. First integration

V4.5 first integrates LLM enrichment into the creative vertical Agent:

```text
Creative Agent deterministic test packages
↓
LLM Gateway
↓
llmTitleVariants
llmMainImageDirections
llmRiskCheck
llmPackagePreviews
```

The deterministic `testPackages` remain valid even when the model is disabled or fails.
