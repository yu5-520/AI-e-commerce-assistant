# CHANGELOG

## V9.9.0

交付验收一致性收口。

- 固定仓库、运行态、API、前端、企业边界、部署验收六段检查流程。
- 新增 V9.9 delivery readiness contract，形成最终交付前的 readiness 证据链。
- 暴露 `/api/architecture/v9/delivery-readiness` 与 `/api/architecture/v9/readiness`。
- 前端缓存版本统一为 `?v=9.9.0`，降低服务器旧页面残留风险。
- 架构守卫升级为 V9.9 运行态入口检查。

## V9.8.0

企业受托运维授权边界。

- 固定高层授权、业务管理、运营执行、外部运维、审计观察五类角色边界。
- 明确外部运维只能执行已授权的系统维护，不能参与经营决策。
- 暴露 `/api/architecture/v9/ops-authorization`。
- 将业务执行权、系统维护权、审批权、审计权拆开，降低私自改数和去痕风险。

## V9.7.0

RAG 审计与回滚一致性。

- 固定 RAG 写入、记忆、审计、回滚的闭环。
- 强化留痕、复盘、权重波动和任务处理记录的可追溯路径。
- 将 RAG 记忆从建议层推进到可审计的企业知识沉淀层。

## V9.6.0

RAG 写入记忆一致性。

- 固定任务结果、复盘结果、运营证据进入 RAG 记忆前的审核路径。
- 避免未审核经验直接污染企业知识库。

## V9.5.0

RAG 命名空间隔离一致性。

- 固定租户、店铺、角色、模块之间的 RAG 数据边界。
- 防止跨租户、跨店铺、跨账号的知识泄露。

## V9.4.0

三层隔离契约一致性。

- 固定前端展示层、后端服务层、数据/RAG 层的隔离边界。
- 将企业级 SaaS 底座从功能堆叠推进到边界治理。

## V9.3.0

前端模块契约一致性。

- 固定总览、经营单元、报表、商品、竞品、上新、流量、待办、日志、系统状态、账号等入口。
- 前端以产品模块表达，避免工程语言挤占操作空间。

## V9.2.0

后端流程一致性。

- 固定报表导入、任务生成、任务同步、审计留痕、复盘沉淀的后端链路。
- 为 SaaS 大厂企业级架构提供统一运行骨架。

## V9.1.0

仓库一致性。

- 收束版本源、README、文档、脚本、API 入口的一致性。
- 将 V9 主目标定位为企业级架构整合底座。
