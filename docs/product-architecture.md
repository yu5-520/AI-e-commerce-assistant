# 产品架构：AI 垂直货架电商经营循环系统

## 1. 统一产品定位

本仓库的主产品定位升级为：

> **AI 垂直货架电商经营循环系统 MVP**

它不是泛电商大而全系统，也不是单点标题 / 主图生成器，而是一个围绕垂直类目商家的 **Workflow-first，Agent-ready** 经营循环原型。

统一叙事：

```text
垂直类目定规则
ERP 管货
CRM 管人
竞品找差距
上新补增长
RPA 做文档任务
日志做复盘
```

旧版“拼多多商品增长 / 商品裂变”不再作为主产品定位，只保留为一个平台案例场景。

## 2. 架构目标

真实货架电商商家的第一需求，通常不是跨类目乱铺，也不是一上来全自动推新，而是先把已有商品经营管起来。

所以系统叙事顺序是：

```text
1. ERP + CRM 经营判断系统
↓
2. 同类目竞品比对系统
↓
3. 同类目上新增长系统
↓
4. 流量测试与数据回流系统
↓
5. 回到 ERP + CRM 经营判断系统
```

核心目标：

1. 以垂直类目知识为底座，避免泛电商空泛判断。
2. 把 ERP / 表格里的商品、订单、库存、退款数据接入工作流。
3. 把 CRM / 客户表里的客户分层、复购、售后、互动记录接入工作流。
4. 从经营问题触发竞品比对，而不是凭空做竞品分析。
5. 从竞品差距和供应链货盘里生成上新、扩品、优化方案。
6. 对小流量测试结果进行回流复盘，重新进入经营判断系统。
7. 用人工审批、日志和报告控制自动化边界。

## 5. 核心工作流

### 5.1 垂直类目配置工作流

```text
选择垂直类目
↓
读取类目知识档案
↓
加载价格带、季节性、售后风险、竞品比对维度
↓
把类目上下文注入经营判断、竞品比对和上新增长
```

不同类目不能共用同一套判断逻辑。

例如防晒服要关注：

```text
季节性 / 面料 / 版型 / 尺码退换 / 防晒卖点 / 价格带 / 主图表达 / 高频差评 / 库存周期
```

### 5.2 ERP + CRM 经营判断工作流

```text
商品表 + 订单表 + 库存表 + 退款表 + 客户表 + 售后记录
↓
商品健康度判断
↓
库存压力判断
↓
毛利与活动价风险判断
↓
退款与售后归因
↓
客户价值分层
↓
输出下一步触发动作
```

经营判断不是终点，而是下一步动作的分流器。

输出示例：

```json
{
  "product_id": "P001",
  "business_status": "库存压力高，转化偏低",
  "risk_level": "medium",
  "trigger_next_step": "competitor_analysis",
  "reason": "同类目商品库存高但订单低，需要比对竞品价格、主图和 SKU 结构"
}
```

### 5.3 同类目竞品比对工作流

竞品比对不是凭空发生，而是由经营问题触发。

```text
经营判断发现问题 / 机会
↓
选定同类目竞品
↓
比对价格带、标题、主图、SKU、评价、差评、售后承诺
↓
判断是优化老品、扩展相似品，还是不建议继续投入
```

触发示例：

```text
点击低 → 比对竞品主图 / 标题
转化低 → 比对竞品价格 / SKU / 评价
退款高 → 比对竞品差评 / 卖点承诺
库存高 → 比对竞品活动价 / 清货打法
类目表现好 → 找相似品扩品
```

### 5.4 同类目上新增长工作流

上新不是盲目铺货，而是从已有经营数据和竞品差距里长出来。

```text
已有经营数据
+
竞品市场差距
+
供应链货盘
↓
新品候选评分
↓
生成标题、主图、SKU、定价、上架字段草案
↓
生成首轮流量测试计划
↓
人工确认后进入测试
```

### 5.5 流量测试与数据回流工作流

```text
新品 / 优化方案上线测试
↓
曝光、点击、转化、退款、ROI、库存变化回流
↓
判断点击问题、转化问题、价格问题、SKU 问题、售后问题
↓
输出继续测试 / 换标题 / 换主图 / 调 SKU / 调价格 / 放量 / 止损
↓
回到 ERP + CRM 经营判断系统
```

### 5.6 RPA 文档任务工作流

RPA 在本产品中优先作为“运营文档和任务流转层”，不是直接操作平台后台。

```text
经营判断 / 竞品比对 / 上新测试结果
↓
生成日报、周报、竞品报告、SKU 建议表、上新检查表、测试复盘报告
↓
进入人工确认
↓
写入审批记录、任务状态和运行日志
```

## 6. 风险边界

MVP 阶段默认保守：

- 不自动上架。
- 不自动改价。
- 不自动报名活动。
- 不自动增加广告预算。
- 不自动群发客户消息。
- 不自动退款。
- 不绕过验证码或平台风控。
- 不生成违规爬虫方案。
- 不保存真实隐私数据。

涉及资金、客户触达、平台后台写入和不可回滚动作时，必须 Human-in-the-loop。

## 9. 架构结论

本产品的核心价值不是“AI 生成得快”，也不是泛泛地承诺“全自动电商”，而是把垂直类目商家的真实经营动作放进一个可解释、可确认、可追踪、可复盘的循环系统里。

最终目标：

> 让商家从“凭感觉看表、改价、上新、测款”升级为“基于垂直类目知识、ERP / CRM 经营判断、同类目竞品比对和测试回流的可控自动化运营”。
