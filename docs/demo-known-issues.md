# 已知问题

## 1. 前端未连接后端

当前 `web_demo/` 是静态页面，展示的是内置样例数据。

解决方向：

- 增加 FastAPI 后端
- 前端调用 `/api/demo/run`

## 2. RAG 不是向量检索

当前 RAG 使用关键词检索模拟。

解决方向：

- 文档切片
- Embedding
- 向量库
- Top-K 检索日志

## 3. AI 节点未接真实模型

当前商品诊断和客户分层使用规则引擎。

解决方向：

- 接入 LLM API
- 保持结构化输出
- 使用 schemas 校验输出

## 4. 没有数据库

当前输出写入本地 `outputs/`。

解决方向：

- SQLite
- workflow_run 表
- approval_record 表
- rpa_execution_log 表

## 5. 没有真实 RPA 执行器

当前只生成任务草案，不执行真实自动化。

解决方向：

- 先实现导出表格 / 报告
- 后续再接 RPA 工具
- 高风险动作仍然人工确认

## 6. 结论

这些是 MVP 阶段的合理边界，不是致命缺陷。

当前项目重点是验证业务闭环，而不是生产级能力。
