# AI 工作流设计

## 1. 总原则

本产品的 AI 工作流不采用“用户输入一句话，大模型直接输出”的简单模式，而是采用结构化流程：

```text
用户输入
↓
信息完整度检查
↓
商品路线判断
↓
风险与规则检查
↓
选择对应 Prompt 模板
↓
生成多个运营版本
↓
结构化输出
↓
保存到商品档案
↓
用户执行并回填数据
↓
AI 复盘并生成下一轮动作
```

核心原则：

- 数据不足时不强行判断。
- 缺少成本时不做利润结论。
- 缺少库存时不建议活动放量。
- 缺少转化数据时不建议强付费。
- 高风险类目先做合规检查。
- 爆品拆解只拆需求结构，不复制侵权素材。

## 2. 工作流一：商品建档与路线判断

### 2.1 输入

- 商品名称
- 类目
- 成本价
- 售价
- 物流成本
- 库存
- SKU
- 商品卖点
- 店铺类型
- 是否品牌 / 授权 / 黑标
- 是否敏感类目
- 竞品信息
- 当前运营目标

### 2.2 AI 判断内容

AI 需要判断：

1. 信息是否完整。
2. 商品是否适合自然流裂变。
3. 商品是否具备爆品测试条件。
4. 商品是否适合强付费放量。
5. 商品是否需要先合规风控。
6. 当前最合适的下一步动作。

### 2.3 输出格式

```json
{
  "recommended_route": "自然流裂变",
  "confidence": "中",
  "reasons": [
    "商品为白牌低客单价商品",
    "当前缺少转化数据，不建议直接强付费",
    "适合先通过标题和主图裂变测试自然流量"
  ],
  "missing_info": ["物流成本", "竞品价格"],
  "next_actions": [
    "生成标题裂变包",
    "生成主图文案裂变包",
    "建立第一轮测试记录"
  ],
  "risk_notes": []
}
```

## 3. 工作流二：标题裂变生成

### 3.1 输入

- 商品档案
- 商品卖点
- 目标人群
- 竞品关键词
- 当前价格
- 当前运营阶段
- 平台风格：拼多多

### 3.2 生成策略

标题不是简单同义词替换，而是按运营意图生成。

标题类型：

- 搜索词覆盖型
- 低价刺激型
- 功能卖点型
- 场景人群型
- 活动承接型
- 长尾关键词型
- 竞品承接型
- 安全保守型

### 3.3 输出字段

每条标题必须包含：

- 标题内容
- 标题类型
- 适合阶段
- 主要关键词
- 测试目标
- 搭配主图方向
- 观察指标
- 更换条件
- 风险提醒

### 3.4 输出示例

```json
{
  "title_versions": [
    {
      "title": "家用厨房置物架免打孔调料架壁挂式卫生间收纳架多功能省空间",
      "type": "搜索词覆盖型",
      "stage": "自然流测试期",
      "keywords": ["厨房置物架", "免打孔", "调料架", "收纳架"],
      "test_goal": "测试搜索词覆盖后的自然曝光",
      "image_match": "功能卖点版主图",
      "metrics": ["曝光", "点击率", "搜索来源成交"],
      "replace_condition": "连续两轮曝光低且点击低时替换"
    }
  ]
}
```

## 4. 工作流三：主图文案裂变生成

### 4.1 输入

- 商品图片
- 商品卖点
- 价格
- SKU
- 竞品主图信息
- 当前运营目标

### 4.2 主图方向

- 低价冲击版
- 功能卖点版
- 痛点解决版
- 场景使用版
- 规格对比版
- 活动承接版
- 竞品差异版

### 4.3 输出字段

每套主图方案必须包含：

- 方案名称
- 主图大字
- 副文案
- 构图建议
- 商品摆放建议
- 搭配标题类型
- 搭配 SKU
- 测试目标
- 风险提醒

### 4.4 输出示例

```json
{
  "image_plans": [
    {
      "name": "低价冲击版",
      "main_text": "多件更划算",
      "sub_text": "厨房 / 卫生间 / 阳台都能用",
      "layout": "商品居中，价格利益点放右上角，使用场景放背景弱化处理",
      "matched_title_type": "低价刺激型",
      "matched_sku": "两件装组合款",
      "test_goal": "测试低价利益点对点击率的提升",
      "risk_notes": ["不要使用虚假原价", "不要写无法兑现的限时承诺"]
    }
  ]
}
```

## 5. 工作流四：SKU 利润测算

### 5.1 输入

- 成本价
- 售价
- 物流成本
- 推广成本预估
- 平台费用预估
- 退款损耗预估
- 活动价
- SKU 组合

### 5.2 计算内容

- 单件毛利
- 活动价毛利
- 保本价
- 可承受推广成本
- 引流款 / 利润款 / 组合款判断
- 是否适合活动
- 是否适合强付费

### 5.3 输出逻辑

如果缺少成本或物流成本：

```json
{
  "status": "insufficient_data",
  "message": "缺少成本或物流成本，无法生成可靠利润测算。"
}
```

如果数据完整：

```json
{
  "sku_strategy": [
    {
      "sku_name": "单件基础款",
      "role": "引流款",
      "price": 9.9,
      "cost": 5.2,
      "shipping_cost": 2.1,
      "gross_profit": 2.6,
      "risk": "利润较薄，只适合测试点击和基础成交"
    }
  ],
  "break_even_price": 8.1,
  "max_ad_cost_per_order": 1.8,
  "recommendation": "建议先用组合款承接利润，不建议单件款强付费放量。"
}
```

## 6. 工作流五：爆品结构拆解

### 6.1 输入

- 爆品标题
- 爆品主图截图
- 爆品价格
- 爆品 SKU
- 爆品活动信息
- 爆品评价 / 差评
- 自家商品信息

### 6.2 分析内容

- 价格带
- 核心关键词
- 主图表达逻辑
- SKU 结构
- 用户购买理由
- 差评机会
- 可承接需求
- 差异化方向
- 侵权风险

### 6.3 输出原则

允许输出：

- 需求结构拆解
- 价格带判断
- 差异化承接方案
- 可测试关键词
- 用户痛点机会

禁止输出：

- 复制对方标题
- 复制对方图片文案
- 使用对方品牌名或商标
- 模仿有明显版权识别的素材
- 规避平台审核的表达

## 7. 工作流六：半付费测试建议

### 7.1 输入

- 当前商品价格
- 成本
- 毛利
- 库存
- 标题版本
- 主图版本
- 点击率
- 转化率
- 推广预算
- 运营目标

### 7.2 输出

- 是否适合半付费
- 建议预算区间
- 测试周期
- 优先测试变量
- 保本线
- 继续条件
- 停止条件
- 下一轮动作

### 7.3 输出示例

```json
{
  "paid_test_status": "small_budget_test_only",
  "budget_suggestion": "先小预算测试，不建议直接放量",
  "test_days": 2,
  "first_variable": "主图点击率",
  "continue_condition": "点击率提升且转化不低于上一轮",
  "stop_condition": "连续两天 ROI 低于保本线且转化无改善",
  "next_action": "保留点击高的主图，再测试不同标题关键词"
}
```

## 8. 工作流七：活动报名准备

### 8.1 输入

- 商品档案
- 当前价格
- 活动价预期
- 库存
- 毛利
- 历史成交
- 退款情况
- 竞品活动价

### 8.2 输出

- 活动适配度
- 建议活动价
- 最低可接受活动价
- 主推 SKU
- 库存建议
- 报名理由
- 活动图文案
- 活动前检查表
- 利润风险
- 活动后复盘表

### 8.3 限制

如果没有库存数据，不建议活动放量。

如果活动价低于保本线，必须提示亏损风险。

如果商品涉及敏感类目，必须先进入合规风控工作流。

## 9. 工作流八：测试复盘与下一轮决策

### 9.1 输入

- 本轮标题
- 本轮主图
- 本轮 SKU
- 本轮价格
- 曝光
- 点击
- 点击率
- 成交
- 转化率
- 推广花费
- ROI
- 退款
- 用户备注

### 9.2 AI 判断

- 曝光是否不足
- 点击是否不足
- 转化是否不足
- ROI 是否低于保本线
- 问题更可能在标题、主图、价格、SKU、详情页还是商品本身
- 下一轮应该继续测、换变量、活动、放量还是止损

### 9.3 输出格式

```json
{
  "round_conclusion": "点击正常，转化偏低",
  "main_problem": "主图能吸引点击，但 SKU 和详情页承接不足",
  "next_actions": [
    "保留当前主图",
    "调整 SKU 组合",
    "详情页前 3 屏增加尺寸、安装、承重说明"
  ],
  "watch_metrics": ["转化率", "客单价", "退款率"],
  "stop_loss_condition": "连续两轮转化无提升且 ROI 低于保本线"
}
```

## 10. Prompt 模板管理

后续开发时，每个工作流应有独立 Prompt 模板：

- route_judgement_prompt.md
- title_variation_prompt.md
- image_plan_prompt.md
- sku_profit_prompt.md
- competitor_analysis_prompt.md
- paid_test_prompt.md
- activity_prepare_prompt.md
- review_decision_prompt.md
- compliance_check_prompt.md

Prompt 输出必须尽量结构化，方便前端展示和后续保存。

## 11. AI 输出要求

所有 AI 输出必须满足：

1. 有明确使用场景。
2. 有测试目标。
3. 有风险提醒。
4. 有下一步动作。
5. 数据不足时不强行给结论。
6. 高风险类目先合规检查。
7. 不鼓励侵权、绕审、虚假宣传。

## 12. 工作流结论

本产品的 AI 工作流要从“生成内容”升级为“运营判断”。

最终目标：

> AI 不只是帮商家写标题，而是帮商家判断这个商品下一轮该怎么测、怎么改、怎么放量，以及什么时候该止损。
