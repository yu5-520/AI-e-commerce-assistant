# 安全与合规复盘

## 1. 安全原则

项目采用保守边界：

> AI 只做判断和建议，RPA 只做低风险任务草案，高风险动作必须人工确认。

## 2. 高风险动作

当前不自动执行：

- 改价
- 上架
- 下架
- 投放
- 报名活动
- 群发客户
- 处理退款
- 诱导好评
- 绕过验证码
- 绕过平台风控

## 3. CRM 隐私边界

当前只使用脱敏 Mock 数据。

不保存：

- 真实姓名
- 手机号
- 微信号
- 地址
- 支付信息
- 身份信息

## 4. RPA 边界

RPA 当前只生成任务草案：

- 日报
- SKU 价格表
- 活动准备表
- 客户分层表
- 售后归因表
- 复盘报告

所有任务默认：

```text
requires_approval = true
auto_execution_allowed = false
```

## 5. Agent 边界

当前不是 Agent-first，不允许 Agent 自主执行高风险动作。

未来只在低风险场景 Agent-ready：

- 数据异常排查
- RAG 检索选择
- 报告生成
- RPA 失败原因分析

## 6. 安全结论

该项目的重点不是追求“全自动”，而是追求：

> 可控、可确认、可追溯、可复盘。
