# V3.0：准实时数据更新 + 报表触发预警

V3.0 将系统从“静态模块 Demo + 手动任务池”升级为“报表导入驱动的经营预警系统”。

## 核心链路

```text
上传 / 导入报表
→ 数据快照 data_snapshot
→ 异常规则判断 alert_event
→ 预警转任务 alert_to_task_bridge
→ 首页、商品页、流量页、报表页、待办、日志、详情报告同步
```

## 新增接口

```text
POST /api/data/import/report
POST /api/data/import/mock-alerts
GET  /api/data/versions
GET  /api/data/versions/latest
GET  /api/data/alerts
GET  /api/data/alerts/entity/{entity_type}/{entity_id}
GET  /api/data/v3-summary
```

## 最小可跑闭环

```text
POST /api/data/import/mock-alerts
→ 从 examples/*.csv 导入库存、退款、订单、商品报表
→ 生成 data_version
→ 生成 alert_events
→ 自动调用 create_task_from_warning()
→ 待办任务池出现报表触发任务
→ 首页 v3 summary 更新
→ 商品页 / 流量页显示 alertState
```

## 设计边界

- V3.0 不直接接淘宝、拼多多、抖音接口。
- V3.0 不自动改价、改库存、调预算、上架商品或触达客户。
- V3.0 先用可解释规则做异常判断，后续 Agent 只补充详情报告和处理清单。
- 重复预警继续走 V2.5.1 的 dedupeKey：活跃任务合并，已归档任务拦截。

## 版本语义

```text
V2.5.1 = 跨账号任务生命周期同步
V3.0.0 = 数据变化触发预警，预警自动进入任务系统
```
