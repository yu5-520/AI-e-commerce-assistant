# V4.4 Feedback Flywheel

V4.4 closes the loop between task handling and RAG memory.

The product rule is:

```text
回流任务 Agent 可以生成经验卡草案和复盘摘要，但不能自动批准入库。
正式 RAG 召回只使用复核通过的结构化经验。
```

## 1. Flow

```text
任务生成 / 模块预警
↓
运营接收与处理
↓
运营提交证据
↓
总管复核通过
↓
自动生成经验卡草案
↓
日报 / 周报回流 Agent 汇总
↓
老板 / 总管确认入库
↓
RAG 召回复核经验
↓
反哺下一轮任务生成、任务解析、标题主图策略
```

## 5. Automatic draft rule

When a manager approves a task, V4.4 adds a pending `feedbackDraft` payload to the task response.

This only creates an experience-card draft. It does not approve the card and does not make it available for formal RAG retrieval until owner / manager review passes.

## 7. Safety boundary

The feedback Agent can:

```text
summarize daily / weekly task outcomes
find learning candidates
generate experience-card drafts
show memory approval metrics
retrieve approved or negative memory as review context
```

It cannot:

```text
automatically approve experience cards
write raw logs directly into formal RAG
skip manager / owner review
directly execute business actions
claim real financial uplift without validated metrics
```
