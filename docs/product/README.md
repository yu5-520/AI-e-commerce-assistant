# Product Docs

本目录只保存产品本体相关文档，不放简历、面试、HR 展示或求职包装内容。

## 文档结构

```text
CHANGELOG.md          产品更新日志：产品定位、主链路、页面/API 边界变更
product-map.md        产品地图：用户、场景、模块、页面结构
domain-model.md       领域模型：核心对象与关系
user-flow.md          用户流程：从导入数据到诊断、审批、报告
module-boundary.md    模块边界：各模块职责与不做什么
mvp-scope.md          MVP 范围：当前阶段必须做、暂不做、后续做
```

## 产品主线

```text
导入经营数据
↓
数据校验与清洗
↓
建立商品档案与客户档案
↓
AI / RAG 经营诊断
↓
生成任务草案
↓
人工确认
↓
导出报告 / 低风险执行
↓
日志回写与复盘
```

## 当前原则

- 先做产品骨架，再做简历包装。
- 先跑通业务闭环，再扩展真实接口。
- 先控制风险边界，再考虑 Agent 自动化。
- 产品主链路、页面结构、API 边界或产品定位发生变化时，必须同步更新 `CHANGELOG.md`。
