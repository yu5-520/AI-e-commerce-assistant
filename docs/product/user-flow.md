# 用户流程

## 1. 设计目标

用户流程用于描述运营人员在产品中的完整操作路径。

当前产品不是聊天框，也不是单次 AI 生成器，而是一个从经营数据进入系统到诊断、任务、审批、报告的闭环工具。

## 2. 总流程

```text
进入工作台
↓
导入经营数据
↓
系统校验数据
↓
建立商品档案与客户档案
↓
运行 AI / RAG 诊断
↓
查看诊断结果和引用依据
↓
生成任务草案
↓
人工确认 / 拒绝 / 修改 / 暂存
↓
导出报告或任务清单
↓
日志回写
↓
下一轮复盘
```

## 3. 流程一：数据导入

### 3.1 用户目标

把商品、订单、库存、退款、客户和互动数据导入系统。

### 3.2 操作路径

```text
打开 /data-import
↓
选择数据类型
↓
上传 CSV / Excel
↓
系统识别字段
↓
用户确认字段映射
↓
系统校验数据
↓
展示错误 / 警告 / 可导入数量
↓
用户确认导入
↓
生成导入记录
```

### 3.3 系统校验

校验内容：

```text
必填字段是否缺失
product_id 是否可关联
customer_id 是否脱敏
价格是否为数字
库存是否为非负数
退款记录是否能关联订单
互动记录是否能关联客户
```

### 3.4 输出结果

```text
导入成功记录
字段映射记录
错误行报告
数据快照
WorkflowRun
ExecutionLog
```

## 4. 流程二：商品诊断

### 4.1 用户目标

判断商品当前经营状态和下一步动作。

### 4.2 操作路径

```text
打开 /products
↓
选择商品
↓
进入 /products/:id
↓
点击“运行商品诊断”
↓
系统读取商品、订单、库存、退款数据
↓
RAG 召回平台规则、合规规则和运营方法
↓
AI / 规则节点生成诊断
↓
展示风险等级、依据和建议动作
↓
用户选择生成任务草案
```

### 4.3 诊断输出

```text
商品经营摘要
SKU 利润风险
库存风险
退款风险
敏感类目风险
建议动作
是否需要人工确认
可生成任务草案
```

## 5. 流程三：客户分层

### 5.1 用户目标

识别高价值客户、沉睡客户、售后敏感客户和流失风险客户。

### 5.2 操作路径

```text
打开 /customers
↓
点击“运行客户分层”
↓
系统读取客户、订单、退款、互动数据
↓
计算 RFM 和售后风险
↓
RAG 召回客户运营方法与客服 SOP
↓
生成客户分层结果
↓
展示客户标签、分层依据和推荐动作
↓
生成客户运营任务草案
```

### 5.3 输出结果

```text
客户分层列表
客户标签
复购建议
售后敏感提示
流失风险提示
是否允许触达
是否需要人工确认
```

## 6. 流程四：售后归因

### 6.1 用户目标

从退款和互动记录中判断售后问题来源。

### 6.2 操作路径

```text
打开 /diagnosis
↓
选择“售后归因”
↓
选择商品或客户群体
↓
系统读取退款、订单、互动和商品数据
↓
RAG 召回客服 SOP 和合规规则
↓
生成售后归因报告
↓
生成商品优化和客服话术草案
```

### 6.3 输出结果

```text
售后问题类型
影响商品
影响客户
数据依据
详情页 / 主图 / SKU / 客服话术优化建议
风险提示
```

## 7. 流程五：任务审批

### 7.1 用户目标

确认、拒绝、修改或暂存由 AI 生成的任务草案。

### 7.2 操作路径

```text
打开 /tasks 或 /approvals
↓
查看 pending_approval 任务
↓
进入任务详情
↓
查看 AI 建议、数据依据、RAG 依据、风险等级
↓
选择确认 / 拒绝 / 修改 / 暂存
↓
系统记录审批结果
```

### 7.3 审批页必须展示

```text
任务类型
目标对象
AI 建议
数据依据
RAG 依据
风险等级
影响范围
是否可回滚
是否允许自动执行
```

### 7.4 输出结果

```text
ApprovalRecord
Task 状态变更
ExecutionLog
```

## 8. 流程六：报告导出

### 8.1 用户目标

拿走可执行的报告、表格或任务清单。

### 8.2 操作路径

```text
打开 /reports
↓
选择报告类型
↓
选择商品 / 客户 / 时间范围
↓
生成报告
↓
预览报告
↓
导出 Markdown / CSV / Excel / JSON
```

### 8.3 输出结果

```text
商品诊断报告
客户分层报告
售后归因报告
SKU 价格建议表
活动准备清单
经营日报
下一轮任务清单
```

## 9. 失败流程

系统必须处理失败情况：

```text
文件格式错误
字段缺失
ID 无法关联
RAG 召回为空
AI 输出格式错误
任务风险过高
用户拒绝审批
导出失败
```

失败后必须展示：

```text
失败原因
影响范围
是否可重试
人工处理建议
下一步操作
```

## 10. MVP 最短闭环

当前最短闭环：

```text
导入 Mock 数据
↓
运行完整诊断
↓
生成任务草案
↓
人工确认 / 拒绝
↓
导出报告
↓
记录日志
```
