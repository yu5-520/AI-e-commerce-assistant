# V9 SaaS 企业级一致性底座

V9 是 SaaS 企业级一致性底座版本。

V9 不新增前端主模块，不继续扩展 V8 权重算法。V9 的目标是把 V1-V8 已有能力统一收束为可商业化、可租户隔离、可私有化部署、可审计、可运维、可长期迭代的企业级 SaaS 架构底座。

## 1. V9 总目标

```text
仓库一致性
前端一致性
后端一致性
数据流一致性
三层套餐隔离一致性
RAG 隔离一致性
权限 / 审计 / 运维一致性
部署模式一致性
测试验收一致性
文档版本一致性
```

V9 的定位：

```text
不是继续造楼层；
而是统一地基、承重墙、电路、水路、消防、门禁和验收标准。
```

## 2. V8 与 V9 的边界

```text
联动比对
交叉验证
审批

```

V9 是底座收束：

```text

```

## 3. 前端一致性

V9 不新增前端主模块。前端主模块保持稳定：

```text
总览
经营单元
商品
竞品
上新
流量
报表
待办
日志
详情页
系统状态
```

V8 权重能力以增强字段的方式进入原模块：

```text
店铺权重 -> 经营单元
商品权重 -> 商品模块
任务强度 -> 待办任务
交叉验证 -> 详情页 / Agent 报告
执行回写 -> 日志 / 复盘
RAG 证据 -> Agent 证据链
```

原则：

```text
不要出现一堆工程化入口。
不要把 V8 暴露成调试按钮。
不要为了新能力重复创建前端模块。
同一模块根据套餐展示不同深度。
```

## 5. 三层套餐隔离一致性

### 5.1 基础版 / Starter

面向中低底部商家，按月付费。

开放能力：

```text
基础报表整理
商品问题分析
商品任务生成
共享脱敏 RAG
基础 Agent 任务草案
```

不开放能力：

```text
商品权重波动算法
店铺权重波动算法
租户私有 RAG
平台趋势深度联动
活动趋势深度联动
复杂审批流
私有化部署
```

### 5.2 专业版 / Professional

面向中高腰部商家，按年付费。

开放能力：

```text
租户隔离数据
租户隔离 RAG
商品权重算法
店铺权重算法
平台趋势
活动趋势
Agent 证据链增强
RAG 数据 / 模板按次维护
```

边界：

```text
客户经营数据必须租户隔离。
客户 RAG 经验库必须租户隔离。
平台趋势和活动趋势可以作为公共增强能力。
RAG 模板单独维修、更改、补充按次数计费。
```

### 5.3 企业版 / Enterprise

面向品牌店、高净值头部店铺。

交付方式：

```text
私有化部署
一次性部署费
年度系统维护费
定制开发费
受托运维服务费
```

开放能力：

```text
部署到客户公司服务器或客户云环境
RAG 数据库放在客户公司存储库
业务数据库放在客户侧
完整店铺 / 商品权重系统
完整审批流
完整执行回写
完整审计日志
高层审批门控
受托运维角色
```

企业版核心不是卖功能，而是卖：

```text
数据安全
权力隔离
经营留痕
审计追溯
系统长期运维
```

## 6. RAG 隔离一致性

RAG 分三层：

```text
shared_rag   共享脱敏知识库，给基础版用
tenant_rag   租户隔离知识库，给专业版用
private_rag  客户私有知识库，给企业版用
```

规则：

```text
所有 RAG 查询必须带 namespace。
所有 Agent 证据必须记录 RAG 来源。
基础版只能读 shared_rag，不能写共享规则。
专业版只能读写本租户 tenant_rag，不能跨租户访问。
企业版 private_rag 部署在客户侧，修改必须走高层授权或受托运维流程。
RAG 留痕默认不可物理删除。
RAG 经验可进入候选，但不能自动改写标准线、权重规则或审批规则。
```

## 7. 权限、审计与受托运维一致性

企业版角色建议：

```text
owner_high_level        高层决策者
business_manager        业务总管
operator                运营执行者
finance                 财务 / 成本角色
audit_observer          审计观察员
external_ops_admin      受托运维
system_service          系统服务账号
```

### 7.1 external_ops_admin 边界

可以做：

```text
系统部署
权限配置执行
RAG 数据库维护
日志备份
审计记录导出
任务流转异常排查
后端版本更新
高层授权后的配置修改
```

不能做：

```text
参与经营决策
审批权重动作
判断商品是否下架
判断投产是否降低
判断运营是否追责
私自删除审计日志
私自修改任务复盘
私自改 RAG 标准线
私自改权重算法
替业务部门提交任务结果
```

### 7.2 四权分离

```text
高层：决策权、审批权、最终解释权
业务部门：任务执行权、数据提交权、结果反馈权
受托运维：系统维护权、配置执行权、日志导出权
系统审计：不可篡改留痕、追溯、复盘记录
```

核心原则：

```text
业务部门没有后端改写权。
运维人员没有经营决策权。
高层不直接手改数据库。
所有关键操作必须走审批流。
所有审批流必须留痕。
所有留痕默认不可物理删除。
```

## 8. 测试验收一致性

V9 后续 smoke test 应拆三套：

```text
基础版 smoke test：
报表导入 -> 商品任务生成 -> 待办 -> 完成 -> 日志

专业版 smoke test：
报表导入 -> 商品 / 店铺权重 -> 交叉验证 -> Agent 证据任务 -> 审批

企业版 smoke test：
私有化配置 -> private_rag -> external_ops_admin -> 审计留痕 -> 高层授权修改
```

CI 不只检查语法，还要检查：

```text
版本一致
路由存在
套餐隔离生效
RAG namespace 生效
权限拒绝生效
任务链路闭环
审计留痕存在
```
