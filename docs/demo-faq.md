# Demo FAQ

## 1. 这个项目是 Agent 吗？

不是深度自治 Agent。

当前项目是 Workflow-first 的 AI 工作流系统：

```text
固定业务流程
↓
AI 在指定节点做判断
↓
人工确认关键动作
↓
RPA 执行低风险任务
↓
日志回写
```

未来可以在低风险环节 Agent-ready。

## 2. 为什么不用一开始就接真实 ERP / CRM？

因为真实 ERP / CRM 涉及账号权限、客户隐私、业务数据安全和平台规则。MVP 阶段先用 Mock 数据验证流程，后续再做接口适配。

## 3. 为什么当前 AI 节点用规则模拟？

因为第一阶段要先验证业务链路、输入输出结构和风险边界。

后续可以把规则节点替换成：

- LLM 调用
- RAG 检索增强
- Function Calling
- MCP 工具调用

## 4. RPA 会自动改价或触达客户吗？

不会。

当前 RPA 只生成低风险任务草案，例如日报、SKU 表、客户分层表、售后归因表。

改价、投放、活动报名、客户触达、退款处理都必须人工确认。

## 5. 这个项目适合简历怎么说？

可以说：

> 围绕电商 ERP / CRM 场景，设计并搭建 Workflow-first 的 AI 自动化运营工作台 MVP，跑通 Mock 数据导入、商品诊断、客户分层、RAG 召回、RPA 任务草案、人工确认和报告输出闭环。

## 6. 当前最适合下一步做什么？

最适合做：

```text
FastAPI 后端接口
↓
前端 fetch 调用接口
↓
任务确认 / 拒绝状态流转
↓
日志回写
```
