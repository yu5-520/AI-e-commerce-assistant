# V7 API 版本用户故事

## 1. 作为演示者

我希望点击前端页面按钮后，页面能调用后端接口并返回真实 Python Workflow 输出。

验收：

- 点击“生成 AI 诊断”后，调用 `/api/demo/run`。
- 页面展示接口返回的商品诊断、客户分层和 RPA 任务草案。

## 2. 作为产品经理

我希望每个 RPA 任务都能展示风险等级、是否需要审批、是否允许自动执行。

验收：

- 任务卡片显示 risk_level。
- 任务卡片显示 requires_approval。
- 任务卡片显示 auto_execution_allowed=false。

## 3. 作为运营人员

我希望可以确认或拒绝一个任务草案。

验收：

- 点击确认后，任务状态变为 approved。
- 点击拒绝后，任务状态变为 rejected。
- 系统记录审批日志。

## 4. 作为安全审核者

我希望系统禁止高风险动作自动执行。

验收：

- 自动改价、自动投放、自动群发、自动退款等任务不会被生成或执行。
- 中高风险任务必须进入 pending_approval。

## 5. 作为面试官

我希望能看到项目从数据、AI、RPA 到前端的完整链路。

验收：

- README 有运行说明。
- Python Workflow 能运行。
- 前端能展示结果。
- Evals 能验证安全边界。
