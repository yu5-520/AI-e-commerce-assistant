# Feedback UI Schema

Frontend should collect feedback with low friction.

## Quick Actions
- copy
- like
- use_this
- rewrite
- not_useful

## Section Feedback
User can give feedback on:
- title_package
- image_package
- sku_suggestion
- price_plan
- operation_action

## Follow-up Feedback
After execution, ask simple questions:
- Did you use it?
- Which item did you use?
- Did exposure, click, conversion, profit, or inventory clearance improve?

## Principle
Do not force long forms. Start with one-click feedback, then optionally collect operation metrics.
